## المقدمة والنطاق

هذه الوثيقة توضح المتطلبات البرمجية الكاملة لموقع إلكتروني خاص بوزارة الاقتصاد والصناعة. يتضمن الموقع:

- **منصة إدارة محتوى** (CMS) لنشر المراسيم والقوانين والأخبار
- **نظام شكاوى إلكتروني ذكي** مع تصنيف آلي
- **مساعد ذكي** (Chatbot) متعدد المنصات
- **محرك بحث دلالي** متقدم
- **نظام أمان متقدم** مع سجل تدقيق كامل

### الهدف الرئيسي
تحسين خدمات الوزارة للمواطنين من خلال:
- توفير معلومات حكومية موثوقة ومحدثة
- توفير قنوات تواصل فعالة للشكاوى والاقتراحات
- تطبيق الذكاء الاصطناعي لتحسين الخدمات
- ضمان الشفافية والحوكمة الإلكترونية

## الوصف العام للنظام

### التكنولوجيات المستخدمة

| المكون | التقنية |
|------|--------|
| Backend | Laravel |
| Frontend | Next.js |
| قاعدة البيانات | PostgreSQL |
| التخزين المؤقت | Redis |
| Server | Ubuntu 22.04 LTS |
| Container | Docker |
| CI/CD | GitHub Actions, Codemagic |

### المستخدمون (Actors)

1. **مدير النظام** - إدارة شاملة للنظام والموظفين
2. **منشئ محتوى** - نشر وتعديل المحتوى الحكومي
3. **مسؤول الشكاوى** - إدارة الشكاوى والمقترحات
4. **المواطنون** - الوصول للخدمات والشكاوى
5. **الزوار** - الوصول للمعلومات العامة

### الوثائق المرفقة
- **دليل المستخدم النهائي لمنشئ المحتوى** - شرح مفصل لإضافة المراسيم وإدارة الأخبار
- **دليل موظف الشكاوى** - الدليل المتكامل لإدارة نظام الشكاوى الذكي
- **الفيديوهات التدريبية** - سلسلة فيديوهات (5-15 دقيقة) عن المهام الأساسية
- **دليل المساعدة المباشرة** - قسم FAQ ومساعد ذكي

## المتطلبات المحددة

### الفرضيات (Assumptions)

- الكادر الوظيفي لديه معرفة أساسية بالتعامل مع متصفحات الويب
- الوزارة لديها مكتب إعلامي مسؤول عن إنشاء المحتوى
- البيانات القديمة (المراسيم والقوانين) من مسؤولية المكتب الإعلامي

### الاعتمادية (Dependencies)

- **خدمات البريد الإلكتروني**: للتحقق الثنائي (2FA) والإشعارات
- **المكتبات البرمجية**: استقرار Laravel و Next.js والتحديثات الأمنية

## حالات الاستخدام (Use Cases)

### FR-01: إنشاء حساب موظف

**الممثلون:** مدير النظام

**الوصف:** إنشاء حسابات جديدة للموظفين وتحديد صلاحياتهم

**الشروط المسبقة:**
- تسجيل دخول مدير النظام إلى لوحة التحكم
- اجتياز التحقق الثنائي (2FA)

**النتائج المتوقعة:**
- إنشاء الحساب في قاعدة البيانات
- إرسال بيانات الدخول آليا للبريد الإلكتروني

**التدفق الرئيسي:**
1. يتوجه المدير إلى قسم "إدارة المستخدمين" ويختار "إضافة مستخدم جديد"
2. يقوم بإدخال البيانات (الاسم الكامل، البريد الإلكتروني، المسمى الوظيفي)
3. يحدد الدور الوظيفي (Role)
4. يتحقق النظام من صحة البيانات وعدم تكرار البريد
5. يضغط على "تأكيد الإنشاء"
6. النظام يوليد كلمة مرور مؤقتة ويخزن الحساب مشفرا
7. إرسال بريد إلكتروني مع رابط التفعيل وبيانات الدخول

### FR-11: نشر محتوى حكومي

**الممثلون:** منشئ محتوى، محرك الذكاء الاصطناعي (ثانوي)، مدير النظام

**الوصف:** نشر المحتوى الحكومي مع الاستعانة بالذكاء الاصطناعي

**التدفق الرئيسي:**
1. اختيار "إضافة محتوى جديد"
2. تحديد نوع المحتوى والجهة التابعة
3. كتابة النص أو رفع الملف
4. طلب تحسين المحتوى بالذكاء الاصطناعي
5. الذكاء الاصطناعي يقترح (عنوان جذاب، كلمات مفتاحية SEO، تصحيح لغوي)
6. المراجعة والقبول أو الرفض
7. النقر على "نشر"
8. تأكيد النشر
9. تحديث الموقع وتسجيل العملية في Audit Trail

### FR-21: إرسال شكوى

**الممثلون:** مواطن/زائر، محرك الذكاء الاصطناعي (ثانوي)

**الوصف:** تقديم شكوى مع التحقق الأمني والتصنيف الآلي

**الشروط المسبقة:**
- الوصول لصفحة الشكاوى
- عدم وجود أكثر من 3 شكاوى في نفس اليوم
- اجتياز Google reCAPTCHA

**النتائج المتوقعة:**
- توجيه الشكوى لوحة تحكم الإدارة المعنية
- إشعار للموظف المختص
- تسجيل في Audit Trail

**التدفق الرئيسي:**
1. اختيار نموذج الشكوى
2. عرض بيانات مقدم الشكوى (غير قابل للتعديل)
3. رفع المرفقات (حد أقصى 5 ملفات، 5MB لكل ملف)
4. اختيار شكوى مرجعية إن وجدت
5. عرض معلومات الشكوى كاملة
6. النقر على "إرسال الشكوى"
7. تأكيد الإرسال
8. التحقق من كونه إنسانا (Google reCAPTCHA)
9. الذكاء الاصطناعي يحلل النص وتحديد الأولوية
10. توليد رقم تتبع عشوائي وفريد

### FR-41: تقديم مقترح/مشروع

**الممثلون:** مواطن/زائر

**الوصف:** تقديم مقترح أو مشروع مع دعم المرفقات

**الشروط المسبقة:**
- الوصول لصفحة المقترحات
- اجتياز Google reCAPTCHA

**التدفق الرئيسي:**
1. الدخول إلى قسم المقترحات والمشاريع
2. اختيار تقديم مقترح/مشروع
3. إدخال البيانات (الاسم، البريد، الهاتف، العنوان، الوصف)
4. رفع المرفقات (حد أقصى 5 ملفات)
5. النقر على زر إرسال المقترح
6. تأكيد الإرسال
7. التحقق من كونه إنسانا
8. التحقق من صحة البيانات
9. حفظ المقترح وتحديد الإدارة المعنية
10. إرسال إشعار للموظف المختص
11. توليد رقم تتبع عشوائي

### FR-28: تعديل حالة شكوى

**الممثلون:** مسؤول الشكاوى، مدير النظام

**الوصف:** تعديل حالة شكوى الواردة

**التدفق الرئيسي:**
1. الدخول إلى لوحة التحكم
2. اختيار قسم الشكاوى الواردة
3. عرض جميع الشكاوى مع إمكانية الفرز
4. الضغط على شكوى لعرض التفاصيل
5. اختيار تعديل الحالة أو الأولوية
6. تأكيد التعديل
7. تسجيل في Audit Trail

### FR-30: الاستعلام عن شكوى (مواطن مصادق عليه)

**الممثلون:** مواطن/زائر

**الوصف:** الاستعلام عن شكوى مقدمة مسبقا

**التدفق الرئيسي:**
1. الدخول إلى الملف الشخصي
2. قسم الشكاوى المقدمة
3. عرض جميع الشكاوى مع حالتها

**التدفق البديل:**
1. الدخول على قناة الواتساب أو التلغرام
2. اختيار استعلام عن شكوى
3. إدخال رقم الشكوى الفريد
4. عرض حالة الشكوى

### FR-31: الاستعلام عن شكوى (زائر)

**الممثلون:** زائر

**الوصف:** الاستعلام عن شكوى بدون تسجيل دخول

**التدفق الرئيسي:**
1. الدخول إلى قسم الشكاوى
2. اختيار استعلام عن شكوى
3. إدخال رقم الشكوى الفريد
4. عرض حالة الشكوى

## واجهات النظام (External Interfaces)

### واجهات الاستخدام (User Interfaces)

**معايير التصميم:**
- معايير عصرية مبنية بـ Next.js
- مكتبة Tailwind CSS لضمان سرعة عالية

**التوافقية:**
- متجاوبة تماما (Responsive)
- تعمل على الحواسب المكتبية والأجهزة اللوحية والهواتف الذكية

**معايير الوصول:**
- الالتزام بـ WCAG 2.1 AA لسهولة الاستخدام

**العناصر القياسية:**
- رأس الصفحة (Header): شعار الوزارة، محرك بحث ذكي، تبديل اللغة (عربي/إنجليزي)
- لوحة التحكم (Admin Dashboard): شريط جانبي مع خيارات حسب الدور الوظيفي
- الوضع الداكن (Dark Mode): خيار التبديل بين الفاتح والداكن

**رسائل الأخطاء:**
- رسائل عامة وواضحة
- منع عرض التفاصيل التقنية الأمنية

### الواجهات المادية (Hardware Interfaces)

- **الخوادم:** خوادم مركز البيانات الحكومي مع Ubuntu 22.04 LTS
- **أجهزة المستخدمين:** لوحة مفاتيح، فأرة، شاشات لمس
- **الشبكة:** بروتوكول TCP/IP

### الواجهات البرمجية (Software Interfaces)

- **قاعدة البيانات:** PostgreSQL مع Laravel
- **التخزين المؤقت:** Redis للـ Caching
- **البيئة:** Docker للعزل والاستقرار
- **المكتبات الخارجية:** OAuth/JWT للمصادقة، PDF generation

### واجهات الاتصال (Communications Interfaces)

- **البروتوكول:** HTTPS مع SSL/TLS 1.2+
- **البريد الإلكتروني:** SMTP لإرسال OTP
- **تشفير البيانات:** AES-256 للبيانات المخزنة
- **تنسيق البيانات:** JSON بين Frontend و Backend
